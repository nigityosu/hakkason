# このゲームの目標
 - 簡単にクリアできるボスにしてください
# userについて
 - userは、ゲームをプレイする人のことを指します
 - ページを開いた瞬間（ゲーム開始時）にuserの名前とパスワードを入力するようにしてください
 - userの名前とパスワードは、ゲームをクリアしたときに表示してください
 - userの名前とパスワードをJSONに記録し、残りのED数を表示してください
 - user情報を受け取る側のJSONは、user_records.jsonを使用してください
 - userの名前とパスワードをJSONに記録し、userの名前とパスワードを入力してゲームを再開できるようにしてください
 - 新規と続きからを選択できるようにしてください
 - 同じ名前で新規を選択されたときは、その名前を持つユーザーがいることを表示してください
# 画面
 - 画面は、主人公の体力と、現在の距離を表示してください
 - 主人公の体力は、HPバーにしてください
 - 現在の距離は、主人公がスタートしてからどれだけ進んだかを表示してください
 - ボスの思い出は、ボスの思い出の数を表示してください
 - 主人公の思い出は、主人公の思い出の数を表示してください
 - ボスの思い出で出たストーリーを一覧表示してください
 - 思い出の上限はなく選択ボタンの下に表示してください
 - resultは、ゲームをクリアもしくはゲームオーバーになったときに表示されるようにしてください
 - result画面は、ゲーム終了時に画面真ん中に出るようにしてください
 - result画面には、エンディングの内容と、集めた思い出の数を表示してください
 - result画面には、もう一度プレイするボタンを表示してください
 - result画面には、パスワードを表示しないでください
 - result画面には、ログアウトボタンを表示してください
# ゲームの内容
    - 道を一直線に進むと、ボスがいます。
    - 横に道を作り、主人公の思い出やボスの思い出を配置してください
    - ボスはそのまま進んでも倒せるようにしてください
    - ボスの思い出は、ボスとの戦いを回避できるようにしてください
    - 主人公の思い出は、15%の確率で主人公の体力を70回復するようにしてください
    - ゲーム中は現在の詳細は表示せず、選択肢のみを表示してください
    - ゲームオーバーの条件は、主人公の体力が0になることとしてください
    - 和解の条件は、ボスの前に行くだけにしてください
    - イベントの数を寄り道すると一回増えるようにしてください
    - 進む距離を寄り道すると一回増えるようにしてください
    - ランダムでイベントが発生するようにしてください
    - 敵発生イベントや味方発生イベントを作ってください
    - ボスの思い出を百個回収すると、進むことしかできないようにしてください
    - 一度見たボスの思い出は、二度と表示されないようにしてください
    - ボスの思い出を回収するたびにボスのストーリーを表示してください
        - ボスの思い出はJSONから取得してください

# クリア条件
 - ボスを倒すこと
 - ボスと和解すること
  - ボスとの和解には、ボスの思い出をそれぞれ1個回収するごとに+5%の確率で成功するようにしてください
 - ボスの思い出の数によって、エンディングが変わるようにしてください
 - 主人公の思い出の数によって、エンディングが変わるようにしてください
